## Key Decisions Made

1. **DB-first queue loading** — All scheduling/quoting endpoints now query DB first for real pending orders. Falls back to demo set only if DB is empty. This reflects actual shop state rather than hardcoded test data.

2. **Ratio-based price assertions** — Tests now compare relative prices (discount percentages) rather than absolute USD amounts. Makes tests resilient to commodity price volatility (steel moves 10-20% week-to-week).

3. **Spot price fallbacks** — No hard dependency on Yahoo Finance. Network failure → conservative defaults (steel, aluminum, etc.). Quote endpoint stays responsive.

4. **Density-based weight** — Volume from STL × material density is more accurate than geometric assumptions. Supports any volume source (STL, manual input, ARIA).
